## 前言

我做一件事，已经找到方法了。但下次再用，还得从头回忆、重新操作。手动复盘太费时间，想让AI帮我把这个过程整理出来，下次直接套用。

---

## 价值

- **安全**：用Git管理每个版本，可随时回溯。AI可能误删文件，历史备份能避免损失（本人经历过4次数据丢失，深有体会）。
- **质量**：把做事的经验固化为步骤，下次按同样流程执行，减少偏差。
- **效率**：一次整理，多次复用，节省重复劳动的时间。

## 快速开始

**前提**：你需要有一段与AI完整跑通的任务对话记录（从目标设定到验收完成的全过程）。

1. **下载“行种”文件**  
   从GitHub仓库下载 `skill.md` 保存到本地。

2. **把“行种”发给AI**  
   在**完成任务的那个AI对话框里**，将 `skill.md` 文件拖入（或粘贴文本）并发送。

3. **AI自动生成新技能**  
   AI会根据当前对话历史，按“行种”规则提炼出结构化步骤并返回。将返回内容保存为 `skill(new).md`（文件名可自定义，方便下次识别即可）。

4. **下次直接用 `skill(new)`**  
   以后做同类任务时，直接把 `skill(new).md` 发给AI，说“按这个执行”，AI就会按步骤自动干活。

**进阶**：用“行种”生成的 `skill(new)` 本身也是技能。你可以继续用“行种”去提炼这个 `skill(new)` 的诞生过程，生成更高阶的 `skill`。

## 常见问题

**Q1：一定要用你推荐的那几个工具吗？**  
A：不用。任何AI对话框都能用，推荐的是我自己试过好用的。人人对话也可以，但需要自己记下对话内容，这部分我还在试，后面会有更新。

**Q2：行种存成什么格式比较方便？**  
A：纯文本就行。记在记事本、备忘录或者GitHub里，下次能找着就可以。

**Q3：AI会不会把我存的文件删掉？怎么防？**  
A：有可能，我自己就遇到过几次。所以建议用Git管理所有行种文件，本地和远程都存一份，即使误删也能恢复。每次操作前顺手备份一下更保险。

**Q4：按行种执行的结果不对怎么办？**  
A：可以改行种里的步骤，或者让AI重做一遍，同时告诉它哪里有问题。行种可以反复修改，直到结果满意为止。

## 附录：获取AI的几种方式

- **第一类：免费工具（国产IDE/CLI）**  
  例如一些国产AI编程助手、命令行工具，直接使用不收费，但生成的技能或对话记录可能无法迁移到其他平台。

- **第二类：免费自建大模型**  
  在自己的电脑或服务器上部署开源大模型（如LLaMA、ChatGLM等）。需要一定的硬件配置和技术能力，模型和数据完全自主可控。

- **第三类：付费API**  
  购买大厂商的API服务，例如OpenAI API，按使用量计费。国内也有类似服务，如智谱AI、百度文心等。适合需要稳定、高性能且不想自己维护的用户。
